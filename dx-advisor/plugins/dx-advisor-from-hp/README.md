# DX Advisor Plugin

企業のホームページURLから、DX/AI導入で効果がでそうな業務を特定し、レポートを自動作成するClaude Codeプラグイン。

---

## クイックスタート

### 1. マーケットプレイスの登録

```
/plugin marketplace add ./ai-homepage
```

### 2. プラグインのインストール

```
/plugin install dx-advisor@ai-homepage
```

### 3. スキルを使う

```
/dx-advisor:assess https://example.co.jp
```

---

## スキル一覧

| スキル | コマンド | 用途 |
|--------|---------|------|
| assess | `/dx-advisor:assess https://example.co.jp` | フルアセスメント（情報収集→分析→レポート） |
| crawl | `/dx-advisor:crawl https://example.co.jp` | 情報収集のみ |
| report | `/dx-advisor:report {企業情報}` | 分析済み情報からレポート作成のみ |

---

## エージェントチーム構成

| エージェント | 役割 | 並列数 |
|-------------|------|--------|
| web-researcher | 企業Webサイトの情報収集 | 3並列 |
| dx-analyst | DX/AI導入機会の分析・評価 | 2並列 |
| report-writer | 経営層向けレポート作成 | 1 |

### 実行フロー

```
Phase 1: 情報収集
┌─────────────────────────────┐
│ web-researcher ×3（並列）    │
│ ├─ #1 企業概要・事業構造    │
│ ├─ #2 業務プロセス・組織    │
│ └─ #3 IT/DX状況・業界動向   │
└─────────────┬───────────────┘
              ↓ 統合
Phase 2: 分析
┌─────────────────────────────┐
│ dx-analyst ×2（並列）        │
│ ├─ #4 フロント・顧客接点    │
│ └─ #5 バック・コア業務      │
└─────────────┬───────────────┘
              ↓ 統合
Phase 3: レポート作成
┌─────────────────────────────┐
│ report-writer ×1            │
│ └─ 経営層向け最終レポート   │
└─────────────────────────────┘
```

## 免責事項

> 本プラグインが提供する分析はAIによる自動分析であり、公開情報から推定した内容を含みます。
> 実際の業務内容や課題とは異なる場合があります。
> 具体的なDX/AI導入検討にあたっては、詳細なヒアリングとPoC（概念実証）を推奨します。
